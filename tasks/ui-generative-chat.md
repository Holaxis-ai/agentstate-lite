## The feature

The launcher (aslite ui) gains a chat box. A prompt like "show me open tasks grouped by
assignee with staleness" produces, within the session, a generated bundle page the user
can click into — the same sandboxed-iframe + v0-bridge architecture every hand-authored
page uses. Generation is conversational: follow-up prompts refine the page.

## Guardrails (constitutive, not decorative)

- The serving agent's capability surface is ALLOWLISTED: the aslite CLI (promote, doc,
  list, kinds) and the bundle-pages authoring pattern (BRIDGE.md contract, embedded
  bridge client). Harness-native output channels — e.g. Claude's Artifacts feature,
  arbitrary file serving, external hosting — are explicitly OUT, so generated UI always
  lands as bundle content: versioned, attributed, synced, sandboxed.
- Generated pages inherit the bridge's read-only-by-construction trust model (no
  mutation messages exist in v0) and the iframe sandbox (opaque origin, connect-src
  'none'). The generator cannot mint a page that exfiltrates or mutates.
- Generated pages are ATTRIBUTED (--actor <human>/genui or similar) and land in a
  distinct namespace (e.g. pages/generated/...) so hand-authored and generated UI are
  distinguishable in the launcher and cleanable in bulk.

## Open design questions (settle in the design pass before building)

1. WHERE THE AGENT RUNS: the ui server spawning a local agent session (which SDK/
   runtime/model)? A separate `aslite genui` verb the launcher shells to? Cost/auth
   model for the underlying LLM calls (local credentials, never bundled).
2. LIFECYCLE: ephemeral drafts vs promoted-to-board; does a draft page sync to
   teammates or stay local until blessed? (Interaction with per-page scoping,
   tasks/ui-pages-per-page-scoping.)
3. LATENCY/UX: streaming generation into the iframe vs generate-then-mint; the
   refine-loop conversation state.
4. RELATION TO THE RETHINK: this IS a concrete answer-shape for roadmap-items/ui-rethink
   ("what replaces the kanban primitive" — answer: pages generated on demand from
   intent). The rethink verdict should adjudicate this feature rather than the feature
   pre-empting the verdict — gate 4 still applies: no build without the explicit human
   decision. This task carries the founders' intent INTO that decision.

## Evidence base

The three hand-authored pages (Board, Roadmap, Memory & Trust — 2026-07-10) prove the
authoring pattern an agent would automate: query/edges over the bridge, self-contained
HTML, promote + registry. The Memory & Trust page was generated by an agent from a
one-line human ask in ~minutes — this feature productizes exactly that loop inside the
launcher.
