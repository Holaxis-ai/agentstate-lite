# Gate

Begin only after the npm CLI + skill prerelease has survived the founder acceptance proof and npm
is the documented primary installation path. Preserve a rollback tag or release artifact before
deleting the old channel.

# Deletion scope

Remove the first-party marketplace distribution channel rather than retaining a permanently thin
duplicate:

- the committed plugin CLI bundle and its shell shim;
- the marketplace/plugin manifests when no longer used by another product;
- plugin-bundle build and byte-drift checks;
- cache-path CLI/reference resolvers and their cross-host/version-precedence tests;
- dual executable regeneration, manifest bumping, and bot workflow machinery;
- skill-bundle-specific branches in invocation/rendering code that the npm-carried skill supersedes;
- stale documentation describing marketplace installation as current.

Retain:

- the npm CLI bundle and installed-package proof;
- the npm-carried generated Agent Skill and declared references;
- explicit skill and hook install/status/uninstall commands;
- bundle-native recipes, references, Kinds, and Views;
- external recipe-folder support through the one RecipeSource pipeline.

# Delivery discipline

Use deletion-focused PRs with direct before/after artifact and command inventories. Do not mix new
product behavior into cleanup. Update the npm verification gate first so every removed marketplace
check has a named surviving owner or is proven obsolete.

# Acceptance

- A clean repository build and npm verification require no marketplace path or committed executable.
- Repository search finds no runtime cache-path discovery for AgentState.
- npm install + optional skill/hook installation still passes the founder journey offline.
- The deleted channel has a documented rollback release, but no live maintenance burden.
- The final line/file reduction is recorded honestly, separating generated bytes from maintained
  source and tests.
